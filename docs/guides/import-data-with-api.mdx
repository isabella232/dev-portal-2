Vertex's Parts Library is a repository of your team's or organization's parts, each with references to geometry and optional metadata.

To import your 3D data into the Parts Library, do the following:

- Create a file resource and upload your 3D data to it. Your files may contain either individual parts or larger assemblies.
- Initiate a translation to create parts from the data contained in the file.

## Uploading files

See our [supported file formats](./supported-file-formats.mdx). If you do not have 3D data handy, download a model from our [3D database](./model-database.mdx) to follow along. To create files in one of these formats, `POST` to the [createFile](https://docs.vertexvis.com/#a8f05429-036c-45b8-89ef-6bd3aa14a7fe) API.

:::note

<ImportAssemblyNote />

:::

<TabItem value="req">

```shell
curl --location --request POST \
  --url "https://platform.vertexvis.com/files" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
  --header "Content-Type: application/vnd.api+json" \
  --data '{
  "data": {
    "attributes": {
      // highlight-next-line
      "name": "vertex-valve.jt"
    },
    "type": "file"
  }
}'
```

</TabItem>
<TabItem value="res">

```json

    "attributes": {
      "created": "2021-01-01T12:00:00.000Z",
      "name": "vertex-valve.jt",
      "status": "pending"
    },
    // highlight-next-line
    "id": "[YOUR_FILE_ID]",
    "type": "file"
  }
}
```

</TabItem>

Now that the file resource exists, `POST` to the [uploadFile](https://docs.vertexvis.com/#0b0a01c2-831d-4b01-87ea-765d4f2e6c1a) API.

  defaultValue="req"
  values={[

<TabItem value="req">

```shell
curl --location --request POST \
  // highlight-next-line
  --url "https://platform.vertexvis.com/files/[YOUR_FILE_ID]" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
  --header "Content-Type: application/octet-stream" \
  --upload-file "[YOUR_PATH_TO_3D_DATA_FILE_INCLUDING_FILE_EXTENSION]"
```

## Creating parts

To make querying for parts easier, Vertex supports supplied IDs. You supply these IDs, likely using existing part and revision IDs from your PLM system. To initiate translations, `POST` to the [createPart](https://docs.vertexvis.com/#05c13b3e-b407-4b19-b110-e6bdc016a714) API. This API is asynchronous, returning the status of a queued translation.

<Tabs
  defaultValue="req"
  values={[
    { label: 'Request', value: 'req', },
    { label: 'Response', value: 'res', },
  ]
}>
<TabItem value="req">

```shell
curl --location --request POST \
  --url "https://platform.vertexvis.com/parts" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
  --header "Content-Type: application/vnd.api+json" \
  --data '{
  "data": {
    "attributes": {
      // highlight-start
      "suppliedId": "[YOUR_SUPPLIED_ID]",
      "suppliedRevisionId": "[YOUR_SUPPLIED_REVISION_ID]"
      // highlight-end
    },
    "relationships": {
      "source": {
        "data": {
          // highlight-next-line
          "id": "[YOUR_FILE_ID]",
          "type": "file"
        }
      }
    },
    "type": "part"
  }
}'
```

</TabItem>
<TabItem value="res">

```json
{
  "data": {
    "attributes": {
      "status": "running"
    },
    // highlight-next-line
    "id": "[YOUR_QUEUED_TRANSLATION_ID]",
    "type": "queued-translation"
  }
}
```

</TabItem>
</Tabs>

To check the status of queued translations, `GET` the [getQueuedTranslation](https://docs.vertexvis.com/#9d4734b1-3300-46bb-bb87-85d23b5e52a3) API. The response returns one of the following:

- The status if `running`.
- An `error` (similar to the response above).
- Upon completion, redirects to the created part.

:::note
The `PART_ID` returned is for the root assembly or part. If your file contains a hierarchy, query for other parts using the [getParts](https://docs.vertexvis.com/#25778d06-038f-410f-bb67-503c932a59a8) API.
:::
