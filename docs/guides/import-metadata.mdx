You can optionally import your team's or organization's metadata properties while importing data and add additional properties to existing part revisions.

## Indexing during part creation

While [creating parts](./import-data.mdx#creating-parts), set the `indexMetadata` request body parameter. This instructs Vertex to extract and index metadata properties from your 3D data.

```json
{
  "data": {
    "attributes": {
      // highlight-start
      "indexMetadata": true
      // highlight-end
      // ...other attributes...
    },
    "relationships": {
      // ...relationships...
    },
    "type": "part"
  }
}
```

## Adding to existing part revisions

To add metadata to existing part revisions, `PATCH` the [updatePartRevision](https://docs.vertexvis.com/#4dc8ffa1-c138-476f-ba9b-d3f1ca382223) API.

<Tabs
  defaultValue="req"
  values={[
    { label: 'Request', value: 'req', },
    { label: 'Response', value: 'res', },
  ]
}>
<TabItem value="req">

```shell
curl --location --request PATCH \
  --url "https://platform.vertexvis.com/part-revisions/[YOUR_PART_REVISION_ID]" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
  --header "Content-Type: application/vnd.api+json" \
  --data '{
  "data": {
    "attributes": {
      "metadata": {
        "MY_STRING_PROPERTY": {
          "value": "PN12345",
          "type": "string"
        },
        "MY_LONG_PROPERTY": {
          "value": "12345",
          "type": "long"
        },
        "MY_FLOAT_PROPERTY": {
          "value": "123.45",
          "type": "float"
        },
        "MY_DATE_PROPERTY": {
          "value": "2021-01-01T12:00:00.000Z",
          "type": "date"
        }
      }
    },
    "id": "[YOUR_PART_REVISION_ID]",
    "type": "part-revision"
  }
}'
```

</TabItem>
<TabItem value="res">

```json
{
  "data": {
    "type": "part-revision",
    "id": "[YOUR_PART_REVISION_ID]",
    "attributes": {
      "created": "2021-01-01T12:00:00.000Z",
      "suppliedId": "[YOUR_SUPPLIED_REVISION_ID]"
    },
    "relationships": {
      "part": {
        "type": "part",
        "id": "[YOUR_PART_ID]"
      }
    }
  }
}
```

## Getting metadata

To get metadata for a part revisions, `GET` the [getPartRevision](https://docs.vertexvis.com/#3260f3e6-8264-4c44-b41d-595a4c37f14a) API. By default, the response does not include metadata. You must explicitly request it using the `fields[part-revision]=metadata` query parameter.

:::note

This API is eventually consistent. If you just added or updated metadata, it may take time for the response to reflect changes.
:::

<Tabs
  defaultValue="req"
  values={[
    { label: 'Request', value: 'req', },
    { label: 'Response', value: 'res', },
  ]
}>
<TabItem value="req">

```shell
curl --location --globoff --request GET \
  --url "https://platform.vertexvis.com/part-revisions/[YOUR_PART_REVISION_ID]?fields[part-revision]=metadata" \
  --header "Authorization: Bearer [YOUR_ACCESS_TOKEN]" \
```

</TabItem>
